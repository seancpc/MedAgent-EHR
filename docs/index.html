醫療 EHR Agentic AI（MedAgent-EHR）

Stanford MedAgentBench ~88%（超越 GPT-4o 72% / Claude 3.5 Sonnet 70%）・完全本地自架・單卡 RTX 4090

設計概念

  • 有別於「問一句答一句」的聊天式 AI，這是 agentic AI：由 LLM 自己規劃步驟、決定呼叫哪個工具、執行臨床任務，並在寫入病歷前自我把關。
  • 不固定路徑、不固定步數的控制流（LLM 驅動，非寫死的流程）。
  • 完全自架於本地 GPU，不依賴任何雲端 LLM API。

專案簡介

系統在標準的 FHIR 電子病歷（EHR）環境中，執行查詢與寫入兩類臨床任務（查最新血糖、記錄血壓、開立檢驗／轉診醫囑等）。

採 Planner–Executor–Verifier 三角色 agent 架構，並以史丹佛的 MedAgentBench（300 題、10 類臨床任務）作為客觀評測，用其官方評分程式打分。

原始碼

→ 完整程式與說明：github.com/seancpc/MedAgent-EHR

系統架構

  • Planner：把臨床任務拆解成有序步驟。
  • Executor（ReAct loop）：逐步呼叫工具、依結果決定下一步。
  • Verifier：寫入病歷前的安全閘，審核每筆 staged write 才允許 commit。
  • 工具透過 MCP 串接 HAPI FHIR R4，讀寫皆走標準 FHIR API。

使用技術

  • LLM：Gemma-4-26B-A4B-it（QAT Q4，Google），以 llama.cpp 自架於 RTX 4090（亦驗證過 Qwen3.6-35B-A3B）
  • Agent：Planner / Executor（ReAct）/ Verifier 三角色
  • 工具與資料：MCP（fhir-mcp-server）、HAPI FHIR R4
  • 寫入安全：兩階段 stage → commit + Verifier 把關 + 醫療碼／劑量驗證
  • 評測：MedAgentBench 官方 refsol grader、Python、Flask（web demo）

成效亮點

  • MedAgentBench 官方評分：~88%（259–263 / 300），高於論文公布最佳 baseline（GPT-4o 72%、Claude 3.5 Sonnet 70%）。
  • 讀取類任務近滿分；寫入類經 FHIR 形狀對齊後 task3／task8 達 30/30。
  • 模型可移植性：因台灣法規將底層 LLM 從 Qwen（中國）遷移至 Gemma-4（Google），因系統與模型解耦，切換幾乎零程式改動、分數近乎持平（265 → 263）。

誠實說明：此 88% 反映的是「為此 benchmark 對齊過的 agent 系統」表現，非模型原始能力的純比較；能分辨「系統表現」與「模型能力」，正是本專案的工程價值之一。task10（條件 + [值,時間] 特殊格式）為部署模型 Gemma 的已知輸出弱項，經調查確認屬模型限制後，選擇保留誠實成績、不對特定題打補丁。

未來可擴展性

  • 以真實藥品碼查詢工具（NDC／SNOMED／LOINC）取代由題目提供代碼，提升真實場景的自主性。
  • 擴展至更多臨床任務類型與多步條件決策。
  • 對接真實 EHR、加入更嚴格的臨床安全規則與稽核軌跡。

開發歷程

  • 問題：寫入任務全數失敗，agent 產生的 FHIR 資源與官方 grader 期望的形狀不符。
  • 解決方式：讓工具接受題目情境提供的代碼／數值／時間，verbatim 帶入 FHIR 資源（code passthrough），task3 由 0→30。
  • 問題：安全閘 Verifier 只看得到一行摘要、看不到完整 staged 資源，誤拒了正確的寫入。
  • 解決方式：讓 staged 摘要帶上完整 FHIR resource，Verifier 能真正檢視內容，task8 由 0→30。
  • 問題：讀取題查不到資料 —— 工具硬把詞彙 resolve 成 LOINC，但資料用的是本地碼（GLU／MG／A1C／K）。
  • 解決方式：get_observations 加 code 參數，用題目給的碼 verbatim 查詢、跳過 resolve，讀取題大幅提升。
  • 問題：用「只重跑失敗題」評估，把被改壞的題當已過略過，造成分數假象（虛高至 274，實際只有 ~249）。
  • 解決方式：改為只信「完整 300 題全跑」作為唯一基準。
  • 問題：遷移至 Gemma 後，模型在 task10 把 reasoning／markdown 洩漏進答案，污染輸出。
  • 解決方式：研究官方文件、用正法關閉 thinking 後仍洩漏，確認屬 Gemma-4-26B 的模型層限制；判斷不該為單一題打補丁（gaming），選擇保留誠實成績。
